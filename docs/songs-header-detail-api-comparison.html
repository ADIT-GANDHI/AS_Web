Songs header & song detail — API vs UI comparison

Companion to songs-listing-ui-comparison.html. Summarises how GET /Api/explore_songs and related calls map to /songs/details/[id] and the global header, plus how to reach the dev server locally.

Local dev server (when localhost does not load)

The app is started with Next bound to all interfaces so you can try several URLs.

Try this URLWhen to use it
http://127.0.0.1:3000 First choice if localhost fails (DNS / hosts / VPN quirks).
http://localhost:3000 Default Next message; should match 127.0.0.1 unless something rewrites hosts.
http://<your-LAN-IPv4>:3000 From another device on the same Wi‑Fi: run ipconfig (Windows) and use the IPv4 address shown for your active adapter.

Command used in this repo: npx next dev --hostname 0.0.0.0 --port 3000 — if port 3000 is already in use, stop the other process or use --port 3001 and open that port instead.

  • Confirm the terminal shows ✓ Ready (or compiled without errors) before opening the browser.
  • Disable VPN or corporate proxy temporarily if the page never finishes loading.
  • Windows Firewall may prompt to allow Node — choose allow for private networks.

Why you still see (201) or broken thumbnails

The browser must reach the same host that serves GET /Api/list and CMS paths like /images/…. That origin is AJAB_API_BASE in lib/ajabEnv.ts.

Env variableRole
NEXT_PUBLIC_AJAB_API_BASE Highest priority. Example: https://ajab.designanddevelopment.in/admin (no trailing slash). Used for /Api/… and for /images/… thumbs.
NEXT_PUBLIC_API_URL If set and AJAB_API_BASE is not, the app strips a trailing /Api/ and uses the remainder as the CMS root (e.g. https://host/admin/Api/ → https://host/admin).

Verify in DevTools: open Network, reload /songs. You should see …/Api/list?… with status 200 and JSON {"status":true,"total":…}. If the request is 404 or returns HTML, the app falls back to mock songs and forces nav count 201. Thumbnails from the list use paths such as /images/TN-….jpg; those are joined to AJAB_API_BASE only for those CMS prefixes — so the base must match the server that actually hosts /images.

Header — SONGS suffix (N)

BehaviourSource
Live count inline, e.g. (211) SongsNavCountContext: set from /Api/list on the songs listing, or a lightweight /Api/list?page=1&limit=1 on song detail. CSS class nav-link--songs-count-live hides the static ::after.
Static design fallback (201) On songs pages only, via CLSongs.css on .nav-link--songs::after, when no live total is in context (e.g. API did not return total).
Non-songs routes No pink count suffix on SONGS (no body:has(.cl-songs-page) / .cld-page).

GET /Api/explore_songs → detail page regions

Implemented in CLSongDetailsPage.tsx. Many fields accept objects (via getText) or strings. HTML-heavy strings are passed through htmlToPlainText for safe plain display.

API / data keys (examples)UI region
Songtitle_transliteration, songTitleTransliteration, umbrellaTitle, umbrellaTitleText, song_title, english_translation Hero title, large title block, video title, version card title fallback chain.
singer_name, singer “sings …” row (uppercase).
poet “poet …” under main title (uppercase).
year, Year, song_year Header meta line with location; version card year in parentheses.
location, song_location Header meta line.
about, meta_description, metaDescription, song_description Description paragraph (truncated + “more”).
songLyricsTranslated, song_lyrics_translated, songLyrics, lyrics, songLyricsNotes Lyrics stanzas (split on blank lines after HTML flattening).
songnotes, song_notes NOTES popup (WavyPaperPopup).
youtube_video_id, youtubeVideoId LiteYouTubeEmbed when a resolvable id is present.
thumbnailUrl, thumbnail_url (relative paths) Prefixed with AJAB_API_BASE for version cards and related thumbs.

Other endpoints on the detail route

EndpointCurrent app behaviour
GET /Api/song_versions Not called for now (staging often returns HTTP 500). The versions strip uses a single card built from the same explore_songs payload so the UI stays stable.
GET /Api/related?song_id=… Related section tabs and list; thumbnails resolved against API base; blurbs flattened from HTML.
